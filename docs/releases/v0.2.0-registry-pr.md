## What to change

Replace the entire existing `- name: weather` entry (the last entry in the
file) with the block below. Three fields change — `description` (updated for
seasons/admin page/refinement), `version`, `url`, `sha256` — but replacing the
whole block is easiest.

```yaml
  - name: weather
    description: >-
      Traveling weather systems and seasons for any GoMud world. Discrete
      fronts move across a zone-adjacency graph crawled from your world's room
      exits, strengthen or die based on the biomes they cross, and render
      per-room (indoor rooms shelter from the storm) through standard room
      mutators plus indoor/outdoor-aware ambient emotes. Per-biome season
      tracks (temperate, monsoon, or fully custom/esoteric) reshape the
      climate through the year. Deterministic seeded simulation; state
      persists across reboots; works on a stock world with zero data
      authoring, and all prose/climate/season mechanics are
      builder-overridable YAML. Player 'weather' command, a web admin page (config, live fronts,

      github.com/GoMudEngine/GoMud-Module-Weather.
```

## PR body

```markdown
Updates the `weather` module entry to v0.2.0.

Everything since v0.1.0: the full seasons system (per-biome tracks, esoteric
season support, seasonal exits/emotes), a web admin page (config with
validation, live fronts, spawn/clear actions), per-room weather refinement
(indoor rooms shelter from the storm; `occupied|all|off` modes), bespoke
gentle weather buffs in a documented 59001+ id range with `BuffOverrides`
remapping, ~30 per-biome ambient lines, a builder-extension guide, and CI
(including an engine-coupled job that tracks GoMud master as drift
early-warning).

Also includes Volte6's v0.1.0 review feedback (module-manager install docs,
Go 1.24 minimum) and a boot self-heal for an engine config-overlay merge
issue we hit during release testing (engine-side fix coming as a separate PR
to the GoMud repo).
```
